I. Descripción del Sistema y Alcance

El Sistema de Información para la Gestión de Reservas del Hostal Apu Qhawarina tiene como objetivo automatizar, centralizar y optimizar el proceso de reservas, registro y control de huéspedes, asegurando una administración eficiente de habitaciones y servicios del hostal.

Alcance del sistema

  • Permitir que los huéspedes realicen reservas en línea o de forma presencial.
  • Facilitar la validación automática de disponibilidad de habitaciones.
  • Registrar datos personales y mantener historial de huéspedes.
  • Controlar el estado y ocupación de las habitaciones en tiempo real.

1.2. Funcionalidades del Sistema

A. Funcionalidades Clave (Obligatorias)

  • Gestión de Reservas: Registrar, modificar, validar o cancelar reservas.
  • Validación y Confirmación: Verifica disponibilidad y genera confirmaciones.
  • Gestión de Habitaciones: Control del estado (disponible, ocupada, mantenimiento).
  • Pre Check-In y Check-In: Registro previo y validación a la llegada.
  • Gestión de Usuarios y Roles: Accesos según tipo de usuario.
  • Generación de Reportes: Emite reportes de ocupación e ingresos.

B. Funcionalidades Secundarias (Opcionales / Futuras)

  • Reservas en línea mediante portal web o aplicación móvil.
  • Notificaciones automáticas vía correo o WhatsApp.
  • Módulo de estadísticas y reportes avanzados.

1.3. Identificación de Usuarios Principales (Actores)

Actor / Usuario Rol dentro del Sistema
Huésped (Cliente) Realiza reservas, pagos y pre check-in.
Recepcionista Administra reservas, valida pagos y realiza check-in/check-out.
Administrador / Gerente Supervisa operaciones, gestiona usuarios y reportes.
Sistema de Reserva (Interno) Automatiza validaciones y notificaciones.

II. Requisitos No Funcionales

Seguridad

El sistema debe garantizar la protección de datos mediante autenticación, encriptación y respaldos automáticos.

Justificación: Evita accesos no autorizados y pérdida de información.

Rendimiento

El sistema debe procesar operaciones comunes en menos de 2 segundos.

Justificación: Mejora la eficiencia y experiencia del usuario.

Usabilidad

Debe contar con una interfaz intuitiva y fácil de usar.

Justificación: Minimiza errores y facilita el aprendizaje.

Compatibilidad

Accesible desde distintos navegadores y dispositivos.

Justificación: Favorece la flexibilidad y disponibilidad.

Mantenibilidad

Fácilmente actualizable y escalable.

Justificación: Facilita futuras mejoras sin afectar el sistema.
